# Mobile Testing Demo

A beginner mobile automation project built with Appium, C#, and NUnit.

## About This Project

This project was created as part of my QA learning journey while preparing to return to software testing.

Before starting this project, I had no hands-on experience with mobile test automation. The goal was to learn the fundamentals of Android testing, understand how Appium works, and gain practical experience setting up and automating a real mobile environment.

Out of all the portfolio projects I have completed so far, this was the most challenging one. Setting up Android Studio, configuring the Android SDK, creating an emulator, connecting Appium, and troubleshooting locator and driver issues required a lot of investigation and persistence.

At the same time, it was one of the most satisfying projects to complete. It reminded me that learning a new technology is often less about writing test code and more about understanding how systems work, solving problems, and gradually building confidence through experimentation.

## Features
- Launch Android applications using Appium
- Connect to an Android emulator
- Handle application dialogs and prompts
- Interact with native mobile UI elements
- Search within the YouTube Android application
- Execute automated tests using NUnit
  
## Technologies Used
- C#
- NUnit
- Appium
- Android Studio
- Android Emulator
- UiAutomator2

## Current Status

This project is intentionally small and focuses on learning the fundamentals of mobile automation.

I still have a lot to learn about mobile testing, Appium frameworks, waits, locator strategies, page object models, and test design. However, this project represents my first successful step into mobile automation and provides a foundation that I plan to continue building on.

## What I Learned
- Android emulator setup and configuration
- Appium server installation and driver management
- Connecting Appium to Android devices
- Locating and interacting with native mobile elements
- Investigating page source and accessibility attributes
- Debugging automation failures and locator issues
- Building and running automated mobile tests with C# and NUnit
